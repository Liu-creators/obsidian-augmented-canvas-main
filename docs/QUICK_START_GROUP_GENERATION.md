# 快速开始：Generate Group with AI

## 5 分钟快速上手

### 前置条件

1. ✅ 已安装 Obsidian Augmented Canvas 插件
2. ✅ 已配置 DeepSeek API 密钥
3. ✅ 打开了一个 Canvas 文件

### 第一次使用

#### 步骤 1：创建源节点

在 Canvas 中创建一个文本节点，输入你想要展开的主题，例如：

```
量子计算入门
```

#### 步骤 2：选择节点并打开菜单

1. 点击选中这个节点（节点会高亮显示）
2. 右键点击节点，或点击节点右上角的三点菜单按钮

#### 步骤 3：点击 "Generate Group with AI"

在菜单中找到 **Generate Group with AI** 按钮（图标是 layers 📚）并点击。

#### 步骤 4：输入你的指令

在弹出的对话框中输入你的指令，例如：

```
请创建一个学习指南，包括基础概念、核心原理、应用场景和学习资源
```

或者更简单的：

```
创建知识卡片
```

#### 步骤 5：等待 AI 生成

- 会出现一个占位符节点显示进度
- AI 流式生成内容（实时更新节点数量）
- 生成完成后自动创建 Group

#### 步骤 6：查看结果 ✨

你会看到：
- 一个新的 Group（带有标题）
- Group 内包含 3-6 个相关节点
- 节点自动排列成整齐的布局
- 从源节点到 Group 有一条连线
- 连线上显示你输入的问题

## 自定义配置

### 调整 Group 外观

1. 打开 **Obsidian 设置 → Augmented Canvas**
2. 滚动到 **Group 生成设置** 部分
3. 调整以下参数：

**默认 Group 颜色**（1-6 或 #hex）：
- `1` = 红色 🔴
- `2` = 橙色 🟠
- `3` = 黄色 🟡
- `4` = 绿色 🟢（默认）
- `5` = 青色 🔵
- `6` = 紫色 🟣

**Group 内节点间距**（默认 40px）：
- 较小值（20-30）= 紧凑布局
- 默认值（40）= 适中
- 较大值（60-80）= 宽松布局

**Group 内边距**（默认 60px）：
- 控制 Group 边框到节点的距离
- 建议保持 40-80px 之间

### 示例配置

**紧凑型**（适合内容密集）：
```
节点间距：30px
内边距：40px
```

**宽松型**（适合长文本）：
```
节点间距：60px
内边距：80px
```

## 故障排除

### Q: 点击按钮后没反应？

**检查清单**：
- [ ] 是否只选择了一个节点？（多选或未选都不行）
- [ ] API 密钥是否已配置？
- [ ] 是否在设置中启用了 Group 生成功能？

### Q: AI 只生成了一个节点？

**可能原因**：
- 问题太简单，只需要一个答案
- AI 没有使用新的 `---[NODE]---` 分隔符格式

**解决方案**：
- 在问题中明确要求："生成 5 个不同的部分"
- 使用更具体的指令："从 A、B、C、D 四个方面分析"
- 插件会自动尝试向后兼容格式（`###` 标题或 `---` 分隔线）

### Q: 节点布局不理想？

**解决方案**：
1. 生成后可以手动拖动调整节点位置
2. 在设置中调整间距参数
3. 下次使用时在问题中指定期望的节点数量（如："创建 4 个卡片"）

### Q: Group 标题不满意？

**说明**：
- Group 标题会从你的问题中自动提取
- 例如："创建关于量子物理的学习指南" → 标题："关于量子物理的学习指南"

**解决方案**：
- 在问题开头使用清晰的主题词
- 生成后可以手动修改 Group 的 label

### Q: 内容质量不够好？

**优化建议**：
1. **丰富源节点内容**：在源节点中提供更多上下文
2. **使用更具体的问题**：而不是泛泛的"解释一下"
3. **调整温度参数**：在设置中降低 temperature（提高一致性）或提高 temperature（提高创造性）
4. **迭代改进**：基于生成的 Group 继续提问和扩展

### Q: 节点中的 Markdown 格式没有显示？

**说明**：
- 插件完全支持节点内容中的 Markdown 语法（**粗体**、*斜体*、列表、代码块等）
- 使用新的 `---[NODE]---` 分隔符可以避免与内容中的 Markdown 符号冲突
- Canvas 会自动渲染节点中的所有 Markdown 格式

**如果格式未显示**：
- 确保 AI 使用了正确的 Markdown 语法
- 检查节点内容是否正确传递（查看源文本）
- 尝试重新生成或手动编辑节点内容

## Markdown 格式支持

### 完整 Markdown 支持

生成的节点内容**完全支持 Markdown 语法**，包括：

- **粗体文本**：使用 `**文本**` 或 `__文本__`
- *斜体文本*：使用 `*文本*` 或 `_文本_`
- 列表：有序列表和无序列表
- 代码块：使用 \`\`\` 包裹代码
- 内联代码：使用 \`代码\`
- 标题：`###`、`##` 等（在节点内部）
- 水平线：`---`（在节点内部）
- 链接：`[文本](URL)`
- 其他 Markdown 特性

### 分隔符格式

为了避免与节点内容中的 Markdown 语法冲突，插件使用特殊的分隔符：

**分隔符格式**：`---[NODE]---`

这个分隔符：
- ✅ 不会与内容中的 `###` 标题冲突
- ✅ 不会与内容中的 `---` 水平线冲突
- ✅ 确保节点内容完整保留所有 Markdown 语法
- ✅ Canvas 会自动渲染所有格式

### 示例

**AI 生成的内容**：
```markdown
第一个节点包含 **重要** 内容和 *斜体* 文本。
- 要点 1
- 要点 2
- 要点 3

---[NODE]---

第二个节点包含 `代码示例` 和更多内容。

### 子标题
- 另一个列表
- 包含 **格式化** 项目
```

**在 Canvas 中的显示**：
- 第一个节点：显示粗体、斜体和列表（格式正确）
- 第二个节点：显示代码、子标题和列表（格式正确）

## 高级技巧

### 技巧 1：链式生成

1. 生成第一个 Group（如"基础概念"）
2. 选择 Group 中的某个节点
3. 再次使用 "Generate Group with AI" 深入展开
4. 构建层级化的知识结构

### 技巧 2：对比分析

在源节点中列出多个对象：
```
Python vs Java vs JavaScript
```

指令：
```
从学习曲线、应用场景、生态系统、性能等方面比较
```

### 技巧 3：多维度分解

使用经典的分析框架：
```
请用 SWOT 分析 [主题]
```
或
```
使用 5W1H 方法分析 [问题]
```

### 技巧 4：组合使用

1. 使用 "Ask question with AI" 生成一个深入分析
2. 选择这个分析节点
3. 使用 "Generate Group with AI" 将分析拆分成多个卡片
